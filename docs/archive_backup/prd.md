## 1. 제품명

**Re:View (리:뷰): The AI-Powered Broadcast QC & Post-Mortem Platform**

## 2. 개요 (Overview)

**제품 비전:**
"모든 방송 제작팀이 기술적 실수와 연출적 성과를 데이터 기반으로 식별하고, 효율적으로 복기(Post-Mortem)하며, 다음 방송의 퀄리티를 지속적으로 향상시킬 수 있는 업계 표준 QC(Quality Control) 플랫폼을 제공한다."

**문제 정의:**
포커 토너먼트 같은 라이브 스포츠/이벤트 방송 제작팀(PD, TD, 오디오, CG)은 방송 후 품질 개선을 위해 리뷰를 진행하지만, 이는 극도로 비효율적입니다.

* **수동적이고 파편화된 리뷰:** PD는 수 시간의 영상을 돌려보며 연출을, 오디오 팀은 따로 오디오를, CG 팀은 따로 그래픽을 확인합니다. 피드백이 통합되지 않고 실수가 반복됩니다.
* **주관적 평가:** "컷이 빨랐다", "오디오가 컸다"는 감상에 의존할 뿐, 객관적 데이터가 부재합니다.
* **기술적 QC의 한계:** 명백한 사고 외에 순간적인 블랙 프레임, 오디오 피크, 자막 오타 등 시청자 경험을 해치는 작은 실수들을 놓치기 쉽습니다.
* **기획 의도 검증의 부재:** "주요 선수를 잘 조명했나?", "방송 템포가 지루하지 않았나?" 같은 기획(스토리텔링) 요소를 검증할 방법이 없습니다.

**솔루션:**
**Re:View**는 방송사 및 프로덕션을 위한 B2B SaaS입니다. 최종 방송본(PGM - Program Feed) 영상을 업로드하면, AI가 **4개의 핵심 모듈**을 통해 영상을 분석합니다.

1.  **비디오/연출 (기술 QC)**
2.  **오디오 (기술 QC)**
3.  **그래픽 (기술 QC)**
4.  **스토리텔링 & 페이싱 (기획 QC)**

AI는 분석 결과를 **'협업형 QC 타임라인'**에 시각화합니다. 제작팀은 AI가 자동 감지한 이슈 지점과 함께, 각자의 파트(연출, 기술, 오디오 등)에 대한 코멘트를 시간 코드(Time-coded) 기반으로 작성하고 공유하며 체계적인 포스트모템을 진행할 수 있습니다.

## 3. 타겟 고객 (Target Audience)

* **Primary (핵심 사용자):**
    * **PD (Program Director):** 방송의 전반적인 연출, 스토리텔링, 템포(Pacing) 검토.
    * **TD (Technical Director):** 카메라 전환(스위칭) 실수, 트랜지션 효과 오류, 기술적 오류 검토.
* **Secondary (전문 사용자):**
    * **오디오 엔지니어:** 오디오 레벨 피크(Clipping), 밸런스, 노이즈 검토.
    * **CG 오퍼레이터 / 그래픽 팀:** 자막 오타, 정보 오류, 그래픽 노출 타이밍 검토.
    * **방송 작가:** 기획 의도, 선수 서사, 해설 밸런스 검토.
* **Tertiary (관리자):**
    * **총괄 프로듀서 (Executive Producer):** 방송 퀄리티 요약 리포트 확인, 팀별/방송별 실수 빈도 트래킹.

## 4. 핵심 기능 (Product Features)

### 4.1. 플랫폼 공통 기능 (P0 - 필수)

| 기능 ID | 기능명 | 설명 |
| :--- | :--- | :--- |
| **PF-01** | **프로젝트 대시보드** | 사용자가 속한 팀의 모든 방송 QC 프로젝트(예: "WSOP 2025 Day 1") 리스트를 관리. |
| **PF-02** | **영상 업로드 및 처리** | PGM 원본 영상을 업로드하면, 백그라운드에서 4대 분석 모듈이 동시에 작동. |
| **PF-03** | **협업형 QC 타임라인** | 제품의 핵심 UI. 영상 플레이어 하단에 4개의 분석 트랙과 사용자 코멘트 트랙이 통합 표시됨. |
| **PF-04** | **타임코드 기반 코멘트** | 영상 재생 중 원하는 시점에 코멘트 작성. `@멘션` (담당자 지정), `#태그` (이슈 분류) 기능 포함. |
| **PF-05** | **QC 리포트 생성** | 모든 AI 감지 마커와 사용자 코멘트를 종합하여 PDF 또는 CSV로 Export (회의 자료용). |

### 4.2. AI 분석 모듈 (Analysis Modules)

#### 모듈 1: 비디오/연출 QC 모듈 (기술)

* **목표:** 카메라 워크, 컷 전환, 화면의 기술적 결함을 자동으로 감지.

| ID | 기능명 | 요구사항 (AI가 감지하는 것) | 우선순위 |
| :--- | :--- | :--- | :--- |
| **V-01** | **블랙/프리즈 프레임 감지** | 0.5초 이상의 블랙 프레임 또는 1초 이상의 프리즈 프레임 구간 감지. (가장 치명적 오류) | **P0** |
| **V-02** | **씬 전환(Cut) 로깅** | 모든 카메라 컷 전환(Hard Cut, Dissolve 등) 지점을 타임라인에 마킹. (TD 리뷰용) | **P0** |
| **V-03** | **단기 컷(Short Cut) 경고** | 1.5초 미만으로 지속된 컷(A->B->C)을 감지. (PD의 '어지러운 연출' 검토용) | **P0** |
| **V-04** | **포커스 이탈(Focus Hunting)** | 카메라 초점이 흐려졌다가 다시 선명해지는(Hunting) 구간 감지. | P1 |
| **V-05** | **과도한 흔들림(Shaky Cam)** | 카메라 흔들림이 임계치를 초과하는 불안정한 샷 구간 감지. | P1 |

#### 모듈 2: 오디오 QC 모듈 (기술)

* **목표:** 오디오 레벨, 밸런스, 노이즈 등 기술적 사운드 품질을 자동으로 감지.

| ID | 기능명 | 요구사항 (AI가 감지하는 것) | 우선순위 |
| :--- | :--- | :--- | :--- |
| **A-01** | **오디오 파형 시각화** | 전체 오디오 파형(L/R)을 타임라인에 시각화. | **P0** |
| **A-02** | **피크(Peak) 감지** | 설정된 임계값(예: -3dB)을 초과하는 오디오 클리핑 발생 지점 마킹. | **P0** |
| **A-03** | **기준 미달(Low Level) 감지** | 설정된 임계값(예: -20dB) 미만으로 오디오가 지속되는 구간(무음/과소) 마킹. | **P0** |
| **A-04** | **채널 밸런스 불균형** | L/R 채널 간 레벨 차이가 6dB 이상 벌어지는 구간 감지. | P1 |

#### 모듈 3: 그래픽(CG) QC 모듈 (기술)

* **목표:** 그래픽 타이밍, 텍스트 오탈자(OCR) 등 정보의 기술적 품질을 감지.

| ID | 기능명 | 요구사항 (AI가 감지하는 것) | 우선순위 |
| :--- | :--- | :--- | :--- |
| **G-01** | **그래픽 이벤트 로깅** | 화면에 그래픽(자막, 통계창)이 나타나고(On) 사라지는(Off) 모든 시점 감지. | **P0** |
| **G-02** | **그래픽 지속 시간 경고** | 그래픽이 너무 짧게(예: 2초 미만) 노출되어 읽기 어려운 구간 감지. | **P0** |
| **G-03** | **텍스트 인식 (OCR)** | 그래픽 영역 내의 텍스트(선수명, 칩 카운트 등)를 추출(Extract)하여 저장. | P1 |
| **G-04** | **데이터 교차 검증 (P2)** | 사전에 업로드한 '선수 명단 DB'와 OCR 텍스트를 비교하여 스펠링 오류 감지. (미래 로드맵) | P2 |

#### 모듈 4: 스토리텔링 & 페이싱 QC 모듈 (기획)

* **목표:** 주요 선수 배분, 해설 밸런스, 방송 템포(Pacing) 등 기획/연출 요소를 분석.

| ID | 기능명 | 요구사항 (AI가 감지하는 것) | 우선순위 |
| :--- | :--- | :--- | :--- |
| **S-01** | **해설 STT (Speech-to-Text)** | 해설자 오디오 트랙을 텍스트로 모두 변환(Transcription). (하위 분석의 기반) | **P0** |
| **S-02** | **'데드 에어' (Dead Air) 감지** | 해설 트랙에서 20초 이상 의미 있는 발언이 없는 '지루한' 구간 감지. | **P0** |
| **S-03** | **핵심 이벤트 감지** | (G-01/G-03 연동) "All-in & Call" 등 특정 키워드 그래픽이 노출된 지점 마킹. (클라이맥스 복기) | **P0** |
| **S-04** | **해설 검색 기능** | STT 데이터를 기반으로 특정 단어(예: "필 아이비")를 검색하고 해당 시점으로 점프. | P1 |
| **S-05** | **선수별 언급 빈도 (차트)** | STT 데이터와 선수 DB를 매칭, 해설진이 각 선수를 언급한 횟수/시간을 Bar Chart로 시각화. | P1 |
| **S-06** | **'고민(Tanking)' 구간 감지** | (G-01 연동) 한 플레이어의 턴이 된 후 1분 이상 다음 액션이 없는 구간 감지. (텐션 구간 복기) | P1 |

## 5. 핵심 사용자 플로우 (Core User Flow)

1.  **[프로젝트 생성]** 방송 종료 후, PD '김'이 "Re:View"에 로그인. "Triton 런던 파이널 테이블" 프로젝트를 생성하고 PGM 영상을 업로드한다.
2.  **[AI 자동 분석]** "Re:View"가 백그라운드에서 영상을 분석, 4개의 QC 트랙(비디오, 오디오, 그래픽, 스토리)에 수십 개의 마커(이슈 지점)를 생성한다.
3.  **[팀 초대 및 비동기 리뷰]** PD '김'이 TD '이', 오디오 엔지니어 '박', CG 오퍼레이터 '최', 작가 '윤'을 프로젝트에 초대한다.
    * **오디오 '박' (Audio):** 로그인 후, [오디오 트랙]을 필터링. AI가 감지한 'A-02: 피크' 마커로 즉시 이동. "01:15:30 - 해설자 마이크 클리핑. 다음부터 리미터 -8dB 설정. #오디오실수 @PD김" 코멘트를 남긴다.
    * **TD '이' (Video):** [비디오 트랙]을 필터링. 'V-01: 블랙 프레임' 마커로 이동해 사고 지점 확인. "00:45:10 - 1초 블랙 프레임. 스위처 렉 확인. #기술오류" 코멘트를 남긴다.
    * **CG '최' (Graphics):** [그래픽 트랙]을 필터링. 'G-02: 짧은 지속 시간' 경고를 확인하고, (P1) 'G-03: OCR' 기능으로 인식된 텍스트를 보며 오타를 수동 검토한다.
    * **작가 '윤' (Story):** [스토리 트랙]의 'S-05: 선수별 언급 빈도' 차트를 확인. "A선수(칩 리더) 언급 120회, B선수(파이널 진출자) 언급 5회. 밸런스 편중 심각. #해설밸런스 @PD김" 코멘트를 남긴다.
4.  **[종합 리뷰 및 회의]**
    * **PD '김' (Direction):** 모든 팀원의 코멘트를 확인. [스토리 트랙]의 'S-02: 데드 에어' 구간과 'S-06: 고민' 구간을 집중적으로 보며, 당시 샷 구성(연출)이 지루하지 않았는지 복기한다.
5.  **[포스트모템]** PD '김'이 'PF-05: QC 리포트 생성' 버튼을 클릭. 모든 이슈가 정리된 PDF를 다운로드하여 주간 제작 회의에서 공식 자료로 사용한다.

## 6. 기술 스택 (Tech Stack - 제안)

* **백엔드:** Python (FastAPI/Django) - OpenCV, Librosa, STT/OCR 라이브러리 연동성.
* **AI / 분석:**
    * **Video:** OpenCV
    * **Audio:** Librosa, FFmpeg
    * **Text (STT/OCR):** Google Cloud Speech-to-Text API, Tesseract/EasyOCR
* **프론트엔드:** React / Vue.js (인터랙티브한 멀티트랙 타임라인 구현 필수)
* **데이터베이스:** PostgreSQL (프로젝트, 사용자, 시간 코드 기반 코멘트 등 관계형 데이터)
* **인프라:** AWS (S3, EC2/ECS, AWS Elemental MediaConvert)

## 7. 수익화 모델 (Monetization Strategy) - B2B SaaS

* **Pro (월/연 구독):** 중소형 프로덕션 대상. (사용자 5명, 월 분석 20시간, 스토리지 1TB 제한)
* **Business (월/연 구독):** 대형 프로덕션, 방송사 대상. (사용자 20명, 월 분석 100시간, P1 기능 포함)
* **Enterprise (별도 문의):** 메이저 방송사(WSOP, EPT 등) 대상. (무제한 사용자/분석, P2 기능 우선 개발, 멀티캠 리뷰 등 커스텀 기능, 전담 기술 지원)

## 8. 성공 지표 (Success Metrics - KPIs)

* **활성화 (Activation):** 신규 가입 팀(프로덕션)이 7일 이내 첫 번째 'QC 리포트'를 생성하는 비율.
* **참여 (Engagement):** 프로젝트당 평균 코멘트 수 (팀원들이 리뷰에 활발히 참여하는가?).
* **리텐션 (Retention):** B2B 구독 갱신율 (Churn Rate).
* **핵심 가치 (Value):** "Re:View 사용 후, 동일한 유형의 기술적/연출적 실수가 감소했는가?" (리포트 내 태그 트렌드 분석 또는 고객 설문).

## 9. 미래 로드맵 (Future Roadmap - V2.0+)

* **V-MTR (멀티캠 리뷰):** PGM뿐만 아니라 모든 개별 카메라 소스(CAM 1~10)를 업로드하여, TD가 자신의 스위칭 결정(PGM)과 다른 앵글을 비교하며 복기하는 기능.
* **G-04 (데이터 교차 검증):** OCR 텍스트와 라이브 데이터(칩 카운트 등)를 연동하여 '정보 오류'까지 100% 자동 감지.
* **S-07 (감성 분석):** 해설자의 STT 데이터에 감성 분석(Sentiment Analysis)을 적용, '흥분', '지루함' 등의 톤을 감지하여 텐션 조절 검토.
* **Live QC:** 방송이 진행되는 *동시에* 스태프가 타임코드를 찍으며 코멘트를 입력하고, 방송 종료 후 PGM 파일과 자동 동기화하는 기능.
